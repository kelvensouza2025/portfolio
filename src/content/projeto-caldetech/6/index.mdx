# Código e Boas Práticas

## Web e Mobile

- Estrutura organizada por funcionalidades (feature-based structure)

- Atomic Design para componentes front-end

## Back-end

- Arquitetura MVC atual com separação de controladores, serviços e modelos

- Implementação de CASL para controle de acesso baseado em papéis (RBAC)

- Separação de módulo de integração com terceiros (ERP Bling)

## Padrões usados

- Princípios de SOLID e Clean Code

- DTOs para transitar dados entre camadas
